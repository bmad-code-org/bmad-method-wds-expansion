# Step 01: Load Context

**Goal:** Load everything needed for full page design compositions — page specs, design system, visual direction, and any wireframes to build upon.

---

## Actions

### 1. Load Page Specifications

Read from `{output_folder}/C-Scenarios/`:
- Complete page structure and content requirements
- User journeys and interaction patterns
- Content copy (headlines, body text, CTAs)
- Image placement requirements

### 2. Load Design System

Read the full design system:
- Color palette (primary, secondary, accent, neutrals)
- Typography scale (font families, sizes, weights, line heights)
- Component patterns (buttons, cards, forms, navigation)
- Spacing and layout tokens
- Border radius, shadow, and elevation tokens

### 3. Load Visual Direction

Read brand and visual direction:
- Brand guidelines (if available)
- Mood board or visual references
- Photography direction
- Illustration style

### 4. Load Wireframes

Check `{output_folder}/E-Assets/wireframes/`:
- Load approved wireframes as structural reference
- Note any wireframe annotations about layout

### 5. Summarize Context

```
Page Design Context:
- Pages to design: [list]
- Design system: [name] — [token count] tokens
- Wireframes available: [count] pages
- Visual direction: [summary]
- Content ready: [yes/no per page]
```
